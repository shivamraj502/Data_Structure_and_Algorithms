What is a Priority Queue? (Definition)
A Priority Queue is an abstract data type (ADT) similar to a normal queue, but with one major difference:

In a normal Queue, elements are processed in FIFO (First-In-First-Out) order — the element that arrives first is removed first.
In a Priority Queue, each element has an associated priority. Elements are served (removed) based on their priority, not on the order they were inserted.

Key Rule: The element with the highest priority is always removed first (dequeued).

If two elements have the same priority, then they are served according to their insertion order (or as per implementation).
Priority can be higher number = higher priority or lower number = higher priority (depends on how you define it).

Example:
Imagine a hospital emergency room:

Patient with heart attack → Priority 10 (highest)
Patient with fracture → Priority 5
Patient with cold → Priority 1

Even if the cold patient arrived first, the heart attack patient will be treated first.
Main Operations in Priority Queue

Operation,Description,Typical Time Complexity (with Heap)
Insert / Enqueue,Add an element with its priority,O(log n)
Extract-Max / Extract-Min (Poll),Remove and return the highest priority element,O(log n)
Peek / Find-Max / Find-Min,Return the highest priority element without removing it,O(1)
Decrease-Key / Increase-Key,Change priority of an existing element,O(log n)
Delete,Remove a specific element,O(log n) or O(n)
IsEmpty / Size,Check if empty or get current size,O(1)

How is Priority Queue Implemented?
Priority Queue is an abstract concept — it doesn't specify how to store data. The most common and efficient implementation is using a Binary Heap:

Min-Heap → Smallest element has highest priority (default in Java PriorityQueue)
Max-Heap → Largest element has highest priority

Other possible implementations (less efficient):

Unsorted Array → Insert O(1), Extract O(n)
Sorted Array → Insert O(n), Extract O(1)
Balanced BST → All operations O(log n)

Note:
Heap is a concrete data structure.
Priority Queue is the interface/behavior.
Heap is the best way to implement Priority Queue for most cases.
Priority Queue in Java
Java provides a built-in class:
Javaimport java.util.PriorityQueue;

PriorityQueue<Integer> minPQ = new PriorityQueue<>();           // Min-Heap (smallest first)
PriorityQueue<Integer> maxPQ = new PriorityQueue<>((a, b) -> b - a); // Max-Heap (largest first)
Common methods:

offer(element) or add(element) → Insert
poll() → Remove and return highest priority
peek() → View highest priority without removing
size(), isEmpty()

You can also store custom objects by implementing Comparable or passing a Comparator.
Real-World Uses and Applications
Priority Queues are used everywhere because many systems need to process "important things first":

Operating Systems
CPU Task Scheduling (higher priority processes run first)
Job scheduling in printers, disk I/O

Graph Algorithms
Dijkstra’s Shortest Path Algorithm (to always pick the next closest node)
Prim’s Algorithm for Minimum Spanning Tree
A* Search Algorithm (used in games for pathfinding)

Data Compression
Huffman Coding (used in ZIP, JPEG, MP3 etc.)

Event-Driven Simulations
Simulating real-world systems (traffic, networks, games)

Networking
Packet scheduling in routers (important packets first)

AI and Games
Best-first search, game AI decision making

Other Common Uses
Hospital triage systems
Load balancing in servers
Merge K sorted lists
Finding top K / bottom K elements
Spam filtering, recommendation systems


Advantages

Efficient for repeatedly getting the "most important" item
O(log n) for insert and remove (very fast)
Simple interface

Disadvantages

Does not support efficient search for arbitrary elements (contains() is O(n) in heap-based PQ)
Not suitable if you need random access
